<a id="会话管理"></a>
## 会话管理

| 命令 | 说明 |
|------|------|
| `/resume` | 打开历史会话列表，选择恢复之前的对话 |
| `/rename <名称>` | 对当前会话重命名 |
| `/clear` | 清空当前对话（保留上下文缓存） |
| `/compact [焦点]` | 压缩上下文释放空间，可指定焦点方向 |
| `/rewind` | 回滚到之前的对话节点（对话和代码都可以回滚） |
| `/branch <名称>` | 从当前对话分叉出新会话（别名 `/fork`） |
| `/export <文件名>` | 导出会话到文件 |
| `/cost` | 查看 token 用量和花费 |
| `/context` | 可视化查看上下文窗口使用情况 |
| `/insights` | 生成会话分析报告 |
| `/memory` | 查看/编辑 CLAUDE.md 和自动记忆 |

<a id="键盘快捷键"></a>
## 键盘快捷键

### 基础操作

| 快捷键 | 作用 |
|--------|------|
| `Enter` | 发送消息 |
| `Ctrl + C` | 取消当前生成 |
| `Ctrl + D` | 退出 Claude Code |
| `Ctrl + L` | 清屏（保留上下文） |
| `Esc` | 中断 Claude 回复 |
| `Esc` + `Esc` | 打开回滚/检查点菜单 |
| `?` | 查看当前终端所有可用快捷键 |

### 编辑 & 输入

| 快捷键 | 作用 |
|--------|------|
| `Shift + Enter` | 换行（不发送消息） |
| `Ctrl + G` | 用外部编辑器打开输入（如 vim/notepad） |
| `Ctrl + V` | 粘贴图片（截图、原型、错误信息） |
| `Ctrl + R` | 搜索命令历史 |
| `Ctrl + S` | 暂存当前提示（稍后恢复） |
| `Ctrl + O` | 切换详细输出/精简模式 |
| `Ctrl + Z` | 挂起 Claude Code（`fg` 恢复） |

### 任务 & Agent

| 快捷键             | 作用           |
| --------------- | ------------ |
| `Ctrl + T`      | 切换任务列表显示     |
| `Ctrl + B`      | 将当前任务发送到后台   |
| `Ctrl + F`（按两次） | 杀掉所有后台 Agent |
|                 |              |

### 模型 & 模式

| 快捷键 | 作用 |
|--------|------|
| `Alt + P` (Meta+P) | 打开模型选择器 |
| `Alt + T` (Meta+T) | 切换扩展思考（thinking） |
| `Alt + O` (Meta+O) | 切换快速模式（Fast Mode） |
| `Shift + Tab` | 循环切换权限模式（见下文） |

### 快速前缀

| 输入 | 作用 |
|------|------|
| `/` | 打开命令菜单 / 输入 slash 命令 |
| `!` | 直接执行 bash 命令（例如 `!git status`） |
| `@` + 路径 | 自动补全文件路径（例如 `@src/`） |

<a id="permission-权限模式"></a>
## Permission 权限模式

Claude Code 在执行操作前会请求你的许可。通过 `Shift+Tab` 循环切换三种模式：

<a id="shifttab-三种模式"></a>
### Shift+Tab 三种模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **default** (默认) | 每次执行工具操作（读文件/写文件/运行命令）都会请求确认 | 日常使用，安全可控 |
| **acceptEdits** | 自动批准文件编辑操作，但执行命令仍需确认 | 编码密集场景，减少确认干扰 |
| **plan** (计划模式) | 只读模式，不能修改文件或执行命令 | 审查代码、设计方案、分析问题 |

切换方法：在输入框中按 `Shift+Tab` 循环切换，状态栏会显示当前模式。

<a id="执行命令时的选项含义"></a>
## 执行命令时的选项含义

当 Claude 要执行命令时，会弹出确认框，你需要输入以下选项：

| 输入 | 含义 | 说明 |
|------|------|------|
| `y` / `Yes` | **批准这次** | 仅允许当前这一次操作 |
| `Y` / `Yes to all` | **全部批准** | 本次会话中相同类型的操作不再询问 |
| `n` / `No` | **拒绝这次** | 拒绝当前操作，Claude 会尝试其他方案 |
| `N` / `No to all` | **全部拒绝** | 拒绝此类所有操作 |
| `s` / `Send` | **批准并发送** | 批准操作，并将结果发送给 Claude |
| `ESC` | **取消** | 取消整个请求 |

> 提示：按 `?` 在确认框中也可以查看所有选项。

<a id="命令行换行"></a>
## 命令行换行

在 Claude Code 的输入框中：

| 操作 | 效果 |
|------|------|
| `Shift + Enter` | **换行** （不发送消息） |
| `Enter` | **发送消息** |
| `Ctrl + G` | 用外部编辑器编辑（可自由换行） |

**详细说明：**
- 单按 `Enter` = 发送消息
- 按住 `Shift` 再按 `Enter` = 插入换行（可以写多行提示词）
- 如果提示词很复杂，推荐按 `Ctrl+G` 打开外部编辑器（如记事本/vim）编写

<a id="常用技巧"></a>
## 常用技巧

1. **快速查看快捷键** — 任何时候按 `?` 显示当前终端支持的快捷键
2. **粘贴图片** — `Ctrl+V` 直接粘贴截图（适用于 UI bug 汇报、设计稿讨论）
3. **快速执行命令** — 输入 `!git status` 直接运行 git 命令
4. **引用文件** — 输入 `@` 后跟路径快速引用项目文件
5. **稍微问题** — 用 `/btw 问题` 提出小问题，不占用主上下文
6. **恢复会话** — `/resume` 可查看和恢复之前的所有会话历史
